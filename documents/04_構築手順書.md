### 2.1 SSH鍵ペアの生成（**重要：デプロイ前に必須**）

Azure VM への SSH アクセスに必要な暗号鍵ペア（秘密鍵・公開鍵）を、ローカルマシン上で事前に生成します。
公開鍵は Bicep パラメータファイルに記載され、VMs の `~/.ssh/authorized_keys` に設定されます。

**⚠️ 重要：** SSH鍵の生成は、すべてのAzureリソースデプロイの**前段階**で行う必要があります。

#### 2.1.1 Windows PowerShell での SSH鍵生成

```powershell
# 1. SSH ディレクトリの確認・作成
$sshDir = "$env:USERPROFILE\.ssh"
if (-not (Test-Path $sshDir)) {
    New-Item -ItemType Directory -Path $sshDir -Force | Out-Null
    Write-Host "SSH ディレクトリを作成: $sshDir"
}

# 2. SSH鍵ペアを生成（4096ビット RSA 暗号化）
# パスフレーズなし（自動デプロイ対応）で生成
ssh-keygen -t rsa -b 4096 -f "$sshDir\id_rsa" -N ""

# 出力例:
# Generating public/private rsa key pair.
# Your identification has been saved in C:\Users\<username>\.ssh\id_rsa
# Your public key has been saved in C:\Users\<username>\.ssh\id_rsa.pub
# ...
# +--[RSA 4096]----+
# |     ...        |
# +----[SHA256]----+

Write-Host "✓ SSH鍵ペアが生成されました"
Write-Host "  秘密鍵: $sshDir\id_rsa"
Write-Host "  公開鍵: $sshDir\id_rsa.pub"
```

#### 2.1.2 不意の上書き防止（既存鍵がある場合）

既に SSH鍵が存在する場合は、確認されます：

```powershell
# 既存ファイルが있는 경우の対応:
# 1) 既存鍵を使用したい場合 → "n"（No）で処理中止、引き続き既存鍵を使用
# 2) 新規鍵で初期化したい場合 → 既存の ~/.ssh/id_rsa を削除してから再実行

Remove-Item "$env:USERPROFILE\.ssh\id_rsa*" -Force  # 既存鍵を削除
# その後、上記の生成コマンドを再実行
```

#### 2.1.3 生成されたSSH公開鍵を確認

```powershell
# 公開鍵の内容を確認・コピー用に表示
$publicKeyContent = Get-Content "$env:USERPROFILE\.ssh\id_rsa.pub" -Raw
Write-Host "=== SSH 公開鍵 ===" -ForegroundColor Green
Write-Host $publicKeyContent
Write-Host ""

# クリップボードにコピー（parameters.json へのペースト用）
$publicKeyContent | Set-Clipboard
Write-Host "✓ 公開鍵をクリップボードにコピーしました（parameters.json へペースト可能）" -ForegroundColor Green
```

**出力例：**
```
ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAACAQCj5gL3p2xY7Q8vK+j3wF4mN... user@hostname
```

**⚠️ 注記：秘密鍵のアクセス権について**
- Windows 10/11 の OpenSSH や Git Bash の `ssh-keygen` は、秘密鍵生成時に **自動的に所有者のみがアクセス可能** な権限を設定します
- 追加の ACL 設定は不要です（管理者権限が必要な手動設定は推奨しません）

#### 2.1.4 生成した公開鍵を parameters.json に自動反映

セクション 2.1 で生成した公開鍵を、PowerShell スクリプトで自動的に `parameters.json` に設定します。
手動でコピー＆ペーストする必要はありません。

**⚠️ 重要：このスクリプトは必ず `Bicep/` ディレクトリで実行してください。**

以下のコマンド**すべて**をコピーして、VS Code の PowerShell ターミナルで Bicep ディレクトリ（`cd Bicep`）から**まとめて貼り付けて実行**してください。

   ```powershell
   cd Bicep
   ```

```powershell
function Update-SSHPublicKey {
    # 前提確認：parameters.json の位置を特定
    Write-Host "現在のディレクトリ: $(Get-Location)" -ForegroundColor Cyan
    
    # parameters.json を探す
    $paramsJsonPath = $null
    if (Test-Path ".\parameters.json") {
        $paramsJsonPath = ".\parameters.json"
        Write-Host "✓ parameters.json を現在のディレクトリで見つけました" -ForegroundColor Green
    } elseif (Test-Path "..\Bicep\parameters.json") {
        $paramsJsonPath = "..\Bicep\parameters.json"
        Write-Host "✓ parameters.json を ..\Bicep ディレクトリで見つけました" -ForegroundColor Green
    } elseif (Test-Path ".\Bicep\parameters.json") {
        $paramsJsonPath = ".\Bicep\parameters.json"
        Write-Host "✓ parameters.json を .\Bicep ディレクトリで見つけました" -ForegroundColor Green
    } else {
        Write-Host "❌ エラー: parameters.json が見つかりません" -ForegroundColor Red
        Write-Host "   以下のいずれかのディレクトリで実行してください：" -ForegroundColor Yellow
        Write-Host "   - Bicep ディレクトリ" -ForegroundColor Yellow
        Write-Host "   - openclaw ルートディレクトリ" -ForegroundColor Yellow
        return
    }

    # 1. 公開鍵ファイルの存在確認
    $publicKeyPath = "$env:USERPROFILE\.ssh\id_rsa.pub"
    if (-not (Test-Path $publicKeyPath)) {
        Write-Host "❌ エラー: SSH公開鍵が見つかりません" -ForegroundColor Red
        Write-Host "   パス: $publicKeyPath" -ForegroundColor Red
        Write-Host "   セクション 2.1 でSSH鍵生成を完了してください" -ForegroundColor Yellow
        return
    }
    Write-Host "✓ SSH公開鍵ファイルを確認: $publicKeyPath" -ForegroundColor Green

    # 2. 公開鍵の内容を読み込む
    $publicKeyContent = Get-Content $publicKeyPath -Raw
    $publicKeyContent = $publicKeyContent.Trim()  # 改行を削除
    Write-Host "✓ SSH公開鍵を読み込みました（長さ: $($publicKeyContent.Length) 文字）" -ForegroundColor Green

    # 3. parameters.json を JSON オブジェクトとして読み込む
    try {
        $paramsJson = Get-Content $paramsJsonPath | ConvertFrom-Json
        Write-Host "✓ parameters.json を読み込みました" -ForegroundColor Green
    } catch {
        Write-Host "❌ エラー: parameters.json の読み込みに失敗しました" -ForegroundColor Red
        Write-Host "   エラー内容: $_" -ForegroundColor Red
        return
    }

    # 4. sshPublicKey パラメータを更新
    $paramsJson.parameters.sshPublicKey.value = $publicKeyContent
    Write-Host "✓ parameters.json の sshPublicKey を更新しました" -ForegroundColor Green

    # 5. 更新した JSON を ファイルに書き込む（見やすい形式）
    try {
        $updatedJson = $paramsJson | ConvertTo-Json -Depth 10
        Set-Content -Path $paramsJsonPath -Value $updatedJson -Encoding UTF8
        Write-Host "✓ parameters.json を保存しました" -ForegroundColor Green
    } catch {
        Write-Host "❌ エラー: parameters.json の書き込みに失敗しました" -ForegroundColor Red
        Write-Host "   エラー内容: $_" -ForegroundColor Red
        return
    }

    # 6. 更新内容の確認（最初の50文字表示）
    $sshKeyDisplay = $publicKeyContent.Substring(0, [Math]::Min(50, $publicKeyContent.Length)) + "..."
    Write-Host ""
    Write-Host "== 設定内容確認 ==" -ForegroundColor Cyan
    Write-Host "パラメータファイル: $paramsJsonPath"
    Write-Host "SSH公開鍵: $sshKeyDisplay"
    Write-Host ""
    Write-Host "✓ parameters.json の設定が完了しました。セクション 2.2 以降に進めます。" -ForegroundColor Green
}

# 関数を実行
Update-SSHPublicKey
```

### 5.3 データディスクのマウント
```bash
# ディスク確認
lsblk

# ⚠️ 注意: 
# - /dev/sdb = OS ディスク（絶対にフォーマットしないこと）
# - /dev/sdc = データディスク（Bicep で追加）

# **既にマウント済みの場合（`lsblk` で `/mnt` に sdc1 が表示）は、以下をスキップして、マウント済み処理を実行**

# データディスクがまだフォーマット・マウントされていない場合のみ実行:

# フォーマット（/dev/sdc はパーティションなし）
sudo mkfs -t ext4 /dev/sdc

# マウントポイント作成
sudo mkdir -p /opt/openclaw
sudo mount /dev/sdc /opt/openclaw

# 永続化（fstab に追加）
sudo bash -c 'echo "/dev/sdc /opt/openclaw ext4 defaults,nofail 0 0" >> /etc/fstab'

# 所有権設定
sudo chown -R openclaw:openclaw /opt/openclaw
```

**既にマウント済みの場合（`lsblk` で `/mnt` に sdc1 が表示）のみ実行:**
```bash
# 既存マウントを確認
df -h | grep /mnt

# /mnt のパーミッション設定（必須）
sudo chown -R openclaw:openclaw /mnt

# ✓ これで完了
```

## 6. Azure Key Vault 設定（Bicep 実装後の手動処理）
### 6.1 Managed Identity へのアクセス権付与
Key Vault は Bicep で作成され、VM のマネージド ID には自動で「シークレットユーザー」権限が付与されます。

### 6.1.1 Key Vault の実際の名前を確認
リソースグループ内に作成されたKey Vault を確認します：
```powershell
# Key Vault の実際の名前を確認（動的取得）
$kvName = $(az keyvault list --resource-group $resourceGroupName --query "[0].name" -o tsv)

if ($kvName) {
    Write-Host "✓ Key Vault 名: $kvName" -ForegroundColor Green
} else {
    Write-Host "❌ エラー: Key Vault が見つかりません" -ForegroundColor Red
    Write-Host "   リソースグループ内のすべてのリソースを確認："
    az resource list --resource-group $resourceGroupName --query "[].{name:name, type:type}" -o table
    return
}

# Key Vault の詳細確認
Write-Host ""
Write-Host "=== Key Vault 詳細情報 ===" -ForegroundColor Cyan
az keyvault show --resource-group $resourceGroupName --name $kvName --query "{name:name, location:location, enableSoftDelete:properties.enableSoftDelete, enablePurgeProtection:properties.enablePurgeProtection}" -o table
```

**⚠️ 重要**: 以降のコマンドで `kv-prd-aiops-001` の代わりに、ここで確認した **実際の Key Vault 名** を使用してください。

### 6.2 シークレットの登録
```powershell
# Azure AI Foundry API キー
az keyvault secret set `
  --vault-name kv-prd-aiops-001 `
  --name "AIFoundry-API-Key" `
  --value "<your-api-key>"

# Slack Bot トークン
az keyvault secret set `
  --vault-name kv-prd-aiops-001 `
  --name "Slack-Bot-Token" `
  --value "xoxb-..."

# SSH 秘密鍵（PEM 形式）
az keyvault secret set `
  --vault-name kv-prd-aiops-001 `
  --name "SSH-Key-TargetVMs" `
  --file <path-to-pem-key>
```

### 6.2.1 シークレット登録の確認
```powershell
# 登録されたシークレット一覧を確認
az keyvault secret list --vault-name kv-prd-aiops-001 --query "[].name" -o tsv
# 以下の3つが表示されることを確認：
#   AIFoundry-API-Key
#   Slack-Bot-Token
#   SSH-Key-TargetVMs

# 各シークレットのメタデータ確認（値は表示されない）
az keyvault secret show --vault-name kv-prd-aiops-001 --name "AIFoundry-API-Key" --query "{name:name, id:id, enabled:attributes.enabled, updated:attributes.updated}" -o table

# 秘密鍵が正しく登録されたか確認
az keyvault secret show --vault-name kv-prd-aiops-001 --name "SSH-Key-TargetVMs" --query "attributes" -o table
```

詳細設計書で要件とされている以下の構成を、Azure Portal で手動設定します：

#### 7.2.1 AI Foundry プロジェクトの作成
1. **Azure Portal へログイン** → リソースグループ `rg-aiops-prod` を選択
2. **AI Foundry Hub** (`hub-prd-aiops-001`) をクリック
3. **Projects** → **+ Create project** を選択
4. プロジェクト詳細を入力：
   - **Project name**: `prj-prd-aiops-001`
   - **Description**: OpenClaw deployment project
   - **Connection**: `hub-prd-aiops-001` を選択
5. **Create** を実行

#### 7.2.2 Claude Sonnet モデルデプロイメント
1. 作成したプロジェクト (`prj-prd-aiops-001`) を開く
2. **Deployments** → **+ New deployment** を選択
3. **Model** セクションで `claude-3-5-sonnet-v2` を検索して選択

   **⚠️ 重要**: Anthropic Claude モデルは Azure AI Foundry で利用可能です。
   - 利用可能地域から `Japan East` を確認
   - モデル名: `claude-3-5-sonnet-v2`（または最新の Claude Sonnet 4.6 相当）

4. **Deployment configuration** を設定：
   - **Deployment name**: `deploy-claude-sonnet`
   - **Content filter policy**: 「Create a new filter policy」を選択
     ```
     Policy name: policy-aiops-custom
     Filters:
       - Violence/Harm: Standard (注: インフラ運用コンテキストで「kill」「privilege」が誤検知されないよう調整)
       - Sexual: Enable
       - Hate: Enable
       - SelfHarm: Enable
     ```
   - **Quota (TPM)**: `30000` (Tokens Per Minute)
   - **Deployment type**: Standard
5. **Create** を実行
