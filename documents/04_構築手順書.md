# 熟練Linuxエンジニア育成（AIエージェント化）プロジェクト 構築手順書

この手順書は、詳細設計書に基づいてAzure上にAIエージェント環境を構築するためのステップを示す。
すべてのAzureリソースは **Bicep テンプレート** による Infrastructure as Code で実装し、構成の再現性と自動化を実現する。

## 1. 前提条件
1. **Azureサブスクリプション**
   - Contributor 以上の権限を持つアカウント
   - 対象リージョン（Japan East）でリソース作成が可能であること

2. **ローカルツール環境**
   - Azure CLI (az) v2.45.0 以上
   - Bicep CLI v0.20.0 以上 (`az bicep install` で取得可能)
   - PowerShell 7.0 以上またはbash（スクリプト実行用）

   - ターゲットSubscription ID とリソースグループ名を確認済み

4. **事前作成リソース**（Bicep 対象外）
   - 既に構築済みの対象Linux VM群（IP アドレス確認済み）
   - 既存 VNet/Subnet（ネットワーク設計が完了している場合は、Bicep でも新規作成可）

### 2.1 Bicep テンプレートの構文チェック
デプロイ前に、Bicepファイルが正しくビルドできることを確認：

```powershell
# テンプレートのビルド確認
az bicep build --file .\main.bicep

# 構文エラーが無ければ、ARM JSONファイルが生成される
# 確認: クリーンアップ不要（ARMファイルは生成されるだけ）
```

### 2.2 parameters.json の設定詳細
`infrastructure/parameters.json` を以下のように編集：

```json
{
  "$schema": "https://schema.management.azure.com/schemas/2019-04-01/deploymentParameters.json#",
  "contentVersion": "1.0.0.0",
  "parameters": {
    "projectName": {
      "value": "aiops"  // 組織に合わせて変更可
    },
    "environment": {
      "value": "prod"   // または "dev", "stg"
    },
    "location": {
      "value": "japaneast"  // 設計どおり（東日本）
    },
    "vnetAddressSpace": {
      "value": "10.0.0.0/16"  // VNet アドレス空間
    },
    "mgmtSubnetAddressSpace": {
      "value": "10.0.1.0/24"  // 管理サブネット（中継サーバー）
    },
    "targetSubnetAddressSpace": {
      "value": "10.0.2.0/24"  // 対象サブネット（既存Linux VM群）
    },
    "vmSize": {
      "value": "Standard_B2ms"  // 設計どおり
    },
    "vmAdminUsername": {
      "value": "azureuser"  // VM管理者ユーザー名
    },
    "sshPublicKey": {
      "value": "ssh-rsa AAAA... (ここにSSH公開鍵を貼り付け)"  // ★ 必須: ローカルの ~/.ssh/id_rsa.pub の内容をコピー
    },
    "logAnalyticsRetention": {
      "value": 30  // ログ保持日数（30日）
    }
  }
}
```

**⚠️ SSH公開鍵の設定方法：**
```powershell
# ローカルで SSH キーペアを生成（初回のみ）
# Windows PowerShell の場合
ssh-keygen -t rsa -b 4096 -f $env:USERPROFILE\.ssh\id_rsa

# キーペアが生成される:
#   ~/.ssh/id_rsa (秘密鍵 - 大切に保管)
#   ~/.ssh/id_rsa.pub (公開鍵 - parameters.json に貼り付け)

# ~/.ssh/id_rsa.pub の内容を確認
Get-Content $env:USERPROFILE\.ssh\id_rsa.pub

# 出力例: ssh-rsa AAAAB3NzaC1yc2E...
# この全行を parameters.json の sshPublicKey に貼り付け
```

## 7. Azure AI Foundry 構築（部分的に手動）
Bicep では基本リソースグループのみ作成。モデルデプロイはマネージドリソースのため、Azure Portal または Azure CLI で実施：

### 7.1 AI Foundry ワークスペース確認
```powershell

  --resource-group $resourceGroupName `
  --name hub-prd-aiops-001
```

### 7.2 Azure Portal での AI Foundry プロジェクト・モデルデプロイメント設定

詳細設計書で要件とされている以下の構成を、Azure Portal で手動設定します：

#### 7.2.1 AI Foundry プロジェクトの作成
1. **Azure Portal へログイン** → リソースグループ `rg-aiops-prod` を選択
2. **AI Foundry Hub** (`hub-prd-aiops-001`) をクリック
3. **Projects** → **+ Create project** を選択
4. プロジェクト詳細を入力：
   - **Project name**: `prj-prd-aiops-001`
   - **Description**: OpenClaw deployment project
   - **Connection**: `hub-prd-aiops-001` を選択
5. **Create** を実行

#### 7.2.2 Claude Sonnet モデルデプロイメント
1. 作成したプロジェクト (`prj-prd-aiops-001`) を開く
2. **Deployments** → **+ New deployment** を選択
3. **Model** セクションで `claude-3-5-sonnet-v2` を検索して選択

   **⚠️ 重要**: Anthropic Claude モデルは Azure AI Foundry で利用可能です。
   - 利用可能地域から `Japan East` を確認
   - モデル名: `claude-3-5-sonnet-v2`（または最新の Claude Sonnet 4.6 相当）

4. **Deployment configuration** を設定：
   - **Deployment name**: `deploy-claude-sonnet`
   - **Content filter policy**: 「Create a new filter policy」を選択
     ```
     Policy name: policy-aiops-custom
     Filters:
       - Violence/Harm: Standard (注: インフラ運用コンテキストで「kill」「privilege」が誤検知されないよう調整)
       - Sexual: Enable
       - Hate: Enable
       - SelfHarm: Enable
     ```
   - **Quota (TPM)**: `30000` (Tokens Per Minute)
   - **Deployment type**: Standard
5. **Create** を実行

#### 7.2.3 プライベートエンドポイント設定（セキュリティ強化）
Bicep で VNet 統合を行いたい場合は、以下の追加リソースを検討：

```powershell
# プライベートエンドポイントの作成（オプション、セキュリティ重要）
# 現在のBicep実装では AzureCloud サービスエンドポイント使用
# VNet との完全な統合には、Microsoft.CognitiveServices/accounts の
# privateEndpointConnections を定義する追加テンプレートが必要
```

**現在の実装レベル**: Bicep で CognitiveServices/accounts を作成済み。
**上級オプション**: プライベートエンドポイント利用時は、別途 Bicep モジュール追加が必要。

## 10. Key Vault へのシークレット登録
### 10.1 マネージドアイデンティティの権限設定
Bicep で作成されたマネージドアイデンティティに対して、Key Vault の読み取りアクセス権を付与：

```bash
# VM のマネージドアイデンティティオブジェクトIDを取得
IDENTITY_ID=$(az vm identity show \
  --resource-group rg-aiops-prod \
  --name vm-aiops-bastion-prd \
  --query principalId -o tsv)

# Key Vault アクセスポリシーを設定
az keyvault set-policy \
  --name kv-prd-aiops-xxx \
  --object-id $IDENTITY_ID \
  --secret-permissions get list
```

### 10.2 SSH 秘密鍵の登録
```bash
# Key Vault 名を確認
az keyvault list --output table | grep aiops

# SSH 秘密鍵をシークレットとして登録
az keyvault secret set \
  --vault-name kv-prd-aiops-xxx \
  --name SSH-Key-TargetVMs \
  --file ~/.ssh/id_rsa

# API キーを登録
az keyvault secret set \
  --vault-name kv-prd-aiops-xxx \
  --name AIFoundry-API-Key \
  --value "your-api-key-here"
```

## 17. トラブルシューティング

| 現象 | 原因 | 対処方法 |
| :--- | :--- | :--- |
| Bicep のビルドエラー | Bicep CLI が古い | `az bicep upgrade` を実行 |
| Key Vault アクセス失敗 | マネージド ID に権限がない | Azure Portal で IAM アクセス制御を確認 |
| OpenClaw が起動しない | Python 依存ライブラリ不足 | `pip install -r requirements.txt` を再実行 |
| Slack 連携が動作しない | Webhook URL が間違っている | Slack Bot トークンと URL を再確認 |
