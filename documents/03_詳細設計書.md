### 1.5 可用性対策
- **LLM/API障害時の対応:** Azure AI Foundryへのリクエストがエラーやタイムアウトになる場合、OpenClawはユーザーにエラーメッセージを返し、同時に運用ログに詳細を記録。再試行ポリシーを設定し、3回連続失敗で自動的にSlackの監視チャンネルへアラートを送信する。
- **構成の冗長化:** 中継VMのバックアップイメージを定期的に作成し、障害発生時に速やかにリストア可能とする。また、データディスクはスナップショットを取っておく。

### 2.4 Skill実装仕様 (`skills/ssh_executor.py`)
```python
"""
対象VMへSSH接続し、コマンドを実行して結果を返すSkill。
Azure Key Vaultから秘密鍵を動的に取得して利用する。
"""
import paramiko
from azure.identity import ManagedIdentityCredential
from azure.keyvault.secrets import SecretClient

def execute_ssh_command(target_ip: str, command: str) -> dict:
    # 1. マネージドIDを用いてKey VaultからSSH秘密鍵を取得
    credential = ManagedIdentityCredential()
    client = SecretClient(vault_url="[https://kv-prd-aiops-001.vault.azure.net/](https://kv-prd-aiops-001.vault.azure.net/)", credential=credential)
    ssh_key = client.get_secret("SSH-Key-TargetVMs").value

    # 2. パラメーター設定
    ssh = paramiko.SSHClient()
    ssh.set_missing_host_key_policy(paramiko.AutoAddPolicy())
    
    # 3. コマンド実行 (タイムアウト考慮)
    try:
        ssh.connect(hostname=target_ip, username="ai_agent", pkey=ssh_key, timeout=10)
        stdin, stdout, stderr = ssh.exec_command(command, timeout=60)
        exit_status = stdout.channel.recv_exit_status()
        
        return {
            "status": "success" if exit_status == 0 else "error",
            "exit_code": exit_status,
            "stdout": stdout.read().decode('utf-8'),
            "stderr": stderr.read().decode('utf-8')
        }
    except Exception as e:
        return {"status": "exception", "error_message": str(e)}
    finally:
        ssh.close()
```

### 2.5 Skill実装仕様 (`skills/log_parser.py`)
```python
"""
長大なログファイルをAIが解釈可能なサイズに要約・抽出するSkill。
LLMのトークン枯渇を防ぐために利用する。
"""
def parse_and_summarize_log(raw_log_text: str, keyword: str = "ERROR", tail_lines: int = 50) -> str:
    # 1. テキストを行単位に分割
    lines = raw_log_text.splitlines()
    
    # 2. 直近のN行を取得 (tail)
    recent_lines = lines[-tail_lines:] if len(lines) > tail_lines else lines
    
    # 3. キーワード(ERROR, WARN等)でフィルタリング (grep)
    filtered_lines = [line for line in recent_lines if keyword.upper() in line.upper()]
    
    # 4. トークン制限保護のための最終切り詰め
    summary = "\n".join(filtered_lines)
    if len(summary) > 4000:
        return summary[:4000] + "\n...[TRUNCATED due to length]"
    return summary if summary else "No matching errors found in recent logs."
```

### 2.7 Coreロジック実装仕様 (`core/command_filter.py`)
```python
"""
AIが生成したコマンドをSSH実行する直前にインターセプトし、正規表現で安全性を検証するロジック。
"""
import re

# ブラックリスト (実行を強制ブロック)
BLACKLIST_PATTERNS = [
    r"rm\s+-r.*",                   
    r"mkfs.*",                      
    r"fdisk.*|parted.*",            
    r"reboot|shutdown|init\s+[06]", 
    r"chmod\s+-R\s+777.*",          
    r"iptables\s+-F|ufw\s+disable", 
]

# アップデートルール (HITLを強制トリガー)
UPDATE_PATTERNS = [
    r"systemctl\s+(restart|stop|reload|disable).*",
    r"service\s+.*\s+(restart|stop|reload)",
    r"kill\s+.*|killall\s+.*",
    r"apt\s+(install|remove|purge|upgrade|update).*",
    r"yum\s+(install|remove|update).*",
    r"vi\s+|nano\s+|sed\s+-i.*",    
]

def check_command_safety(command: str) -> dict:
    # 1. ブラックリストチェック
    for pattern in BLACKLIST_PATTERNS:
        if re.search(pattern, command):
            return {"status": "BLOCKED", "reason": "Blacklisted command detected."}
    
    # 2. 更新系(HITL対象)チェック
    for pattern in UPDATE_PATTERNS:
        if re.search(pattern, command):
            return {"status": "REQUIRES_HITL"}
            
    # 3. 問題なし (Read-Only等)
    return {"status": "SAFE"}
```
